--- title: Token Generation Process --- {% extends "columns.j2" %} {% block main %} ## Getting Started
Binstar build is currently in development, you must install the binstar client from GitHub. 
conda config --add channels binstar
conda install binstar-build
binstar config --set url http://api.alpha.binstar.org
### Why Tokens? Tokens can be used to make specific private repositories, collections or packages available to other users. You have total control over the scope of access. ### How To Generate a Token At present, tokens can only be generated using the binstar client. You can find more information on installing and configuring the binstar client [here](/build_cli.html). Once you have successfully installed the binstar client, you can generate a token via the following command: 
  
  binstar auth --create --name MyTokenNameHere --scopes 'repos conda:download'
This will generate a random, alphanumeric token string. Any binstar user to whom you distribute this token will be able to download any of your private conda repositories, of both the user and collection type. To use the token, simply issue the following command: 
  
  conda config --add channels https://conda.binstar.org/t/TokenValue/
### Reviewing Tokens You can review all tokens generated with your binstar account from the command line: 
  
    binstar auth --list
Or in your [user settings](https://binstar.org/settings/access) on [http://binstar.org](http://binstar.org). Please note: although you can review tokens from the binstar client, for security reasons you can only view the actual token value by logging into your binstar.org account. ### Revoking Tokens You can revoke a token at any time from your [user settings](https://binstar.org/settings/access). ### Additional Configuration Options
-o
Set the token owner (must be an organization). Note:You must have administrative rights to an organization in order to use this option.
-s
Scopes for token. For example if you want to limit this token to conda downloads only you would use --scopes "repo conda:download". For a full list of scoping options, use "binstar auth --list-scopes"
-w
Create a shorter token. Note: Default behavior is a longform (stronger) token.
Use the binstar auth -h option for additional configuration information. {% endblock %}
