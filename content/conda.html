For the purposes of this example, we'll use the [conda_gc_test](https://github.com/Binstar/binstar_client/tree/master/example-packages/conda). Therefore: $ cd ~/ $ git clone https://github.com/Binstar/binstar_client $ cd binstar_client/example-packages/conda/ In this directory we can see two required files: [meta.yaml](https://github.com/Binstar/binstar_client/blob/master/example-packages/conda/meta.yaml) and [build.sh](https://github.com/Binstar/binstar_client/blob/master/example-packages/conda/build.sh) (for Linux or Mac) or **bld.bat** (for Windows). From that moment we are ready to build our package: $ cd ../ $ conda build conda/ BUILD START: conda_gc_test Package: conda_gc_test-1.2.1-py27_3 no source + echo Building Building number of files: 0 BUILD END: conda_gc_test-1.2.1-py27_3 Nothing to test for: conda_gc_test-1.2.1-py27_3 # If you want to upload this package to binstar.org later, type: # # $ binstar upload /home/USERNAME/anaconda/conda-bld/linux-64/conda_gc_test-1.2.1-py27_3.tar.bz2 # # To have conda build upload to binstar automatically, use # $ conda config --set binstar_upload yes All packages built in this way are placed in a subdirectory of [Anaconda's](https://store.continuum.io/cshop/anaconda/) *conda-bld* directory. Moreover, please consider sharing your own conda recipes with the community and make a pull request to the official [conda-recipes repository](https://github.com/conda/conda-recipes). {% endcall %} {% call section('conda-installing') %} ## Installing Conda packages The basic way to install a package from [binstar.org](https://binstar.org/) is using `conda`: ``` $ conda install --channel https://conda.binstar.org/REPOSITORY-PATH PACKAGE ``` Where **REPOSITORY-PATH** may be one of the following: 1. A **username**, this allows installation of packages from a user repository: ``` $ conda install --channel https://conda.binstar.org/USERNAME PACKAGE ``` 1. A **user channel**, this allows installation of packages from a collection of packages a user has put together: ``` $ conda install --channel https://conda.binstar.org/USERNAME/channel/CHANNELNAME PACKAGE ``` The **main** channel is the default channel, so: `https://conda.binstar.org/USERNAME/channel/main` is equivalent to `https://conda.binstar.org/USERNAME`. **Note**: All binstar `urls` can omit `https://conda.binstar.org/` part. So, the above installation command can be easily written in short form: $ conda install --channel USERNAME PACKAGE Or if you would like to specify a channel as well: $ conda install --channel USERNAME/channel/CHANNELNAME PACKAGE So now, if You would like to install before uploaded package, please run: $ conda install --channel USERNAME conda_gc_test The following packages will be downloaded: package | build ---------------------------|----------------- conda_gc_test-1.2.1 | py27_3 807 B USERNAME The following NEW packages will be INSTALLED: conda_gc_test: 1.2.1-py27_3 USERNAME Proceed ([y]/n)? y Fetching packages ... conda_gc_test- 100% |######################################################| Time: 0:00:00 119.89 kB/s Extracting packages ... The names and locations of the configuration files vary slightly across platforms. * On Unix and Mac OS X the configuration file is: `$HOME/.condarc` * On Windows, the configuration file is: `%HOME%\.condarc` The preferable way to modify default conda settings is to use its `config` subcommand. For example to add your own binstar acount to be known by conda, you should run: $ conda config --add channels http://conda.binstar.org/USERNAME Again, the short version: $ conda config --add channels USERNAME See also the [customization of .condarc file section](http://conda.pydata.org/docs/faq.html#customization). {% endcall %} {%endblock%}
