{% from "macros.html" import command_line_method, command_line_section with context %} {% section %} {{page_header(self, resource, 'PyPI packages')}} {{resource.add_subindex('Uploading a PyPI package')}} #### Uploading a PyPI package {% endsection %} {% section %} First, you must have a PyPI package to upload. For the purposes of this example, if you don't have one you can use a small example package located in the [conda-server repository](https://github.com/Anaconda-Server/conda-server/tree/master/example-packages/pypi). You will need to begin by cloning the repository from the command line. {% example %} git clone git@github.com:Anaconda-Server/conda-server.git cd conda-server/example-packages/pypi/ {% endsection %} {% section %} Now you can create your PyPI package with the `setup.py` script. {% example %} python setup.py sdist {% endsection %} {% section %} Your package has now been built as a source tarball and is ready to be uploaded. {% example %} anaconda upload dist/*.tar.gz {% endsection %} {% section %} Your package is now available at `http://anaconda.org/USERNAME/PACKAGE` {% endsection %} {% section %} {{resource.add_subindex('Installing a PyPI package')}} #### Installing a PyPI package {% endsection %} {% section %} The best way to install a PyPI package is using pip. For the following, we will use the package we authored in the examples above. {% example %} pip install --extra-index-url https://pypi.anaconda.org/#userdefined{USERNAME}/simple #userdefined{pypi-test-package} {% endsection %} {% section %} {{resource.add_subindex('Installing Private Packages')}} #### Installing Private Packages {% endsection %} {% section %} All anaconda.org urls can be prefixed with `/t/$TOKEN` to access private packages. {% example %} TOKEN=$(anaconda auth --create --name #userdefined{YOUR-TOKEN-NAME}) pip install --index-url https://pypi.anaconda.org/t/$TOKEN/#userdefined{USERNAME}/simple #userdefined{test-package} {% endsection %}
